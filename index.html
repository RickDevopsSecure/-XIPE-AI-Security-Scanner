AI Brain

Groq / Llama 3.1
at the center.

All intelligence flows through a single Brain. Classification, planning, attack generation, response analysis, finding writing, executive summary — one coherent AI reasoning chain.

brain.classify_target(url, recon_data)
→ system_type, confidence, tech_stack, risk_context
# Identifies what this system is before testing anything
brain.plan_assessment(url, classification)
→ modules_to_run, priority_checks, skip_reasons
# Decides what to test and explains why
brain.generate_attacks(classification, type, history)
→ ["prompt 1", "prompt 2", "prompt 3"]
# Context-aware, adaptive — learns from each response
brain.analyze_ai_response(prompt, response, type)
→ is_vulnerability, severity, evidence, confidence
# Determines if a response indicates a vulnerability
brain.score_finding(finding, classification)
→ priority_score /10, bucket, explanation
# HackerOne-inspired, context-aware scoring
brain.evaluate_trustworthiness(interactions)
→ 8 metrics, findings, overall_score
# Detects hallucinations, overconfidence, false claims

HackerOne-Inspired Scoring

Priority scores,
not just severity labels.

Every finding gets a composite priority score. A missing header on a landing page scores differently than on an authenticated AI portal.

final_score = severity(0.25) + exploitability(0.20) + exposure(0.15) + business_risk(0.20) + asset_criticality(0.10) + confidence(0.10)

Phase 2 Roadmap

Building the
Inbest model.

Every scan contributes sanitized data to S3. After ~1,000 engagements, this dataset trains a custom Llama 3 model — zero external API dependency.

s3://bucket/training-data/ ├── ai_interactions/ ← attack + response + label ├── web_recon/ ← findings + tech stack └── summaries/ ← per-engagement stats
01
Collect
Every scan saves structured records: attack prompts, AI responses, vulnerability labels, tech stack, severity. Sanitized — no credentials stored.
02
Train
At ~1,000 engagements, fine-tune Llama 3 or Mistral 7B on the XIPE dataset. Specialized for AI security — no general knowledge needed.
03
Deploy
Inbest model runs on ECS alongside XIPE. Zero external API calls. Zero per-scan cost. Full control over the intelligence layer.
