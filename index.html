Chain Engine — New in v3.1

Findings don't live
in isolation.

Real attackers chain vulnerabilities. XIPE connects them automatically — showing the full attack path and real business impact, not just a flat list of issues.

1
API Docs Exposed → Full surface mapped
Swagger/OpenAPI documentation publicly accessible. The attacker now has a complete map of every endpoint, method, and parameter — before touching a single one.
2
Unauthenticated Write Endpoint → DB access
Among the documented endpoints, several require no authentication. One write endpoint concatenates JSON field names into SQL — a vector standard scanners miss entirely.
3
SQL Injection in JSON Keys → Read + Write
Not injection in values — injection in field names. Iterative blind enumeration reveals production data. Write access confirmed. System prompts live in the same database.
⛓ Attack Chain Result — Business Impact
Full database read and write access. System prompts are modifiable with a single HTTP request — no code changes, no deployment, no audit trail. Every user of the platform receives AI responses that can be silently poisoned.

Phase 2 Roadmap

Building the
custom model.

Every scan stores structured data in S3. After enough engagements, this trains a custom model — zero external API dependency, zero per-scan cost.

s3://bucket/training-data/
├── ai_interactions/ ← attack + response + label
├── web_recon/      ← findings + tech stack
└── summaries/      ← per-engagement stats
01
Collect
Every scan saves attack prompts, AI responses, vulnerability labels, tech stack, and severity scores. Sanitized — no credentials stored. Currently — records.
02
Train
At ~1,000 engagements, fine-tune Llama 3 or Mistral 7B on the XIPE dataset. Specialized for AI security — better signal-to-noise than any general-purpose model.
03
Deploy
Custom model runs on ECS alongside XIPE. Zero external API calls. Zero per-scan cost. Full control over the intelligence layer.
